# Шахматы в браузере (C++ движок + WebAssembly)

Шахматный движок на C++ (папка `chess/`) с полноценным веб-интерфейсом
(папка `web/`), скомпилированным в WebAssembly через Emscripten/Embind.
Готово к развёртыванию на GitHub Pages через GitHub Actions.

## Возможности интерфейса

- Человек против человека, человек против бота, бот против бота.
- Выбор цвета за человека (в режиме «человек — бот»), переворот доски.
- Настраиваемая задержка между ходами бота, пауза автопроигрывания.
- Отмена хода (в игре с ботом «откатывает» и ответ бота, возвращая к
  решению игрока) и произвольная навигация по истории ходов кликом по
  списку или стрелками.
- Продолжение партии новой веткой после отката назад — старая ветка
  при этом отбрасывается.
- Сдача партии (для каждой стороны, управляемой человеком).
- Подсветка последнего хода, доступных ходов выбранной фигуры, шаха и мата.
- Анимация перемещения фигур, звуковые эффекты (ход, взятие, шах,
  превращение, мат) — синтезируются на лету через Web Audio API, без
  внешних аудиофайлов.
- Импорт/экспорт позиции в формате FEN.
- Сохранение текущей позиции и настроек в localStorage (полная история
  ходов не хранится, как и не требовалось по ТЗ).
- Адаптивная вёрстка для мобильных устройств, интерфейс на русском языке.

Правила троекратного повторения, 50 ходов и недостаточности материала
движком не реализованы (как и оговорено) — соответственно, в интерфейсе
эти исходы также не отслеживаются.

## Структура репозитория

```
chess/                  Движок (без изменений в логике)
  src/                   Исходный код движка — как было
  wasm/
    api.hpp / api.cpp    Тонкая обвязка над Board/BotPlayer:
                         принимает/возвращает FEN и JSON-строки,
                         не дублирует и не изменяет логику движка
    bindings.cpp         Embind-биндинги, публикующие функции api.* в JS
  CMakeLists.txt         Добавлена цель chess_wasm (только при сборке
                         под Emscripten) и опциональные тесты (Catch2)

web/                     Статический веб-интерфейс
  index.html
  style.css
  app.js                 Вся логика интерфейса (без сторонних шахматных
                         библиотек — вся легальность ходов проверяется
                         вызовами в движок через wasm)
  engine/                Сюда при сборке кладутся сгенерированные
                         chess.js и chess.wasm (в репозитории их нет —
                         генерируются при сборке)

.github/workflows/deploy-pages.yml
                         CI: установка Emscripten, сборка chess_wasm,
                         публикация web/ на GitHub Pages

build_wasm.sh            Скрипт для локальной сборки (если нужно
                         проверить сборку у себя, а не только через CI)
```

## Как это работает

`chess/wasm/api.cpp` — единственный новый код, взаимодействующий с
движком. Он не переопределяет и не дублирует правила игры: каждый вызов
(`getStatus`, `makeMove`, `getBotMove`) создаёт `Board` из переданного
FEN и делегирует всё движку (`getAllLegalMoves`, `isMoveLegal`,
`makeMove`, `getGameOutcome`, `BotPlayer::makeMove` и т.д.), после чего
формирует JSON-ответ. Дополнительно там же — строгая валидация формата
входного FEN, чтобы посимвольный разбор в `Board::importFromFEN` не мог
получить данные, выходящие за границы строки.

Так как движок не поддерживает отмену ходов, фронтенд хранит FEN после
каждого хода и просто пересоздаёт позицию нужного хода. Продолжение
партии из середины истории отбрасывает «будущую» ветку — ровно как
описано в задании.

## Сборка и деплой на GitHub Pages

1. Включите GitHub Pages в настройках репозитория: **Settings → Pages →
   Source → GitHub Actions**.
2. Запушьте изменения в ветку `main` (или запустите workflow вручную —
   вкладка **Actions → Сборка и деплой на GitHub Pages → Run workflow**).
3. Workflow сам установит современный Emscripten, соберёт
   `chess_wasm` через CMake, скопирует `chess.js`/`chess.wasm` в
   `web/engine/` и опубликует содержимое `web/` как статический сайт.

Сами скомпилированные `chess.js`/`chess.wasm` не хранятся в репозитории
— они пересобираются заново при каждом деплое.

## Локальная сборка и запуск

Требуется установленный [Emscripten SDK](https://emscripten.org/docs/getting_started/downloads.html):

```bash
git clone https://github.com/emscripten-core/emsdk.git
cd emsdk && ./emsdk install latest && ./emsdk activate latest
source ./emsdk_env.sh
cd /путь/к/этому/репозиторию
./build_wasm.sh
python3 -m http.server --directory web 8080
```

Затем откройте `http://localhost:8080` (открывать `index.html` напрямую
через `file://` нельзя — ES-модули и WebAssembly требуют загрузки по
http/https).

## Обычная (нативная) сборка движка без веба

Поведение `CMakeLists.txt` вне Emscripten не изменилось: собирается
`minmax_chess` (консольная версия из `main.cpp`), а `tests` подключаются
автоматически, если в системе найден Catch2 (`find_package(Catch2 3
QUIET)` вместо `REQUIRED` — чтобы конфигурация не падала при отсутствии
Catch2, если тесты не нужны).

```bash
cmake -S chess -B build
cmake --build build
```
